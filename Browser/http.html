    # 一次完整的请求过程

    1. 首先用户输入url。
    2. 浏览器拿到url地址后进行dns解析。
    3. 三次握手与服务器建立连接。
    4. 发送http请求。
    5. 服务器返回响应。
    6. 四次挥手与服务器断开连接。

    # HTTP 协议特点

    1. 无状态协议
    2. 简单可扩展

    # Method

    安全的：不会修改服务器数据的方法 GET HEAD OPTIONS

    幂等：同样的请求被执行一次与连续执行多次的效果是一样的，服务器状态也是一样的。所有安全的方法都是幂等的。 GET HEAD OPTIONS PUT DELETE

    # 状态码

    状态码 意义
    1xx 表示请求已接受
    2xx 请求成功
    3xx 重定向
    4xx 客户端出现错误
    5xx 服务端错误

    1. 更快、更稳定、更简单

    # HTTPS和HTTP的对比

    HTTPS 比 HTTP 多了一层TSL/SSL加密。

    对称加密：使用相同的密钥进行加密解密

    非对称加密：发布公私钥，私钥服务器保留。私钥加密只有公钥能解，公钥加密只有私钥能解。

    # 跨域问题

    什么是跨域？

    同源策略：一个网站只能访问相同来源的数据。当两个域名的scheme + host name + port三者都相同时，可认为两个域名为相同的来源。

    如果访问了其他域中的数据，就被称为跨域。

    # 用户鉴权

    由于HTTP是一种无状态的协议，所以服务器并不知道用户的历史请求情况。如果我们需要记住用户的登录状态，需要采用一些特殊的方式，这就是用户鉴权。

    Cookie

    cookie 是不可跨域的（可通过domain实现资源共享）

    优点：可以实现用户鉴权。

    缺点：存储在客户端，安全性不高。

    Session

    优点：比Cookie更安全，且可存储任意类型的数据。

    缺点：容易受到CSRF的攻击。Session的存储时间一般较短。

    Token

    优点：可以在移动端使用。由于Token是一种无状态的鉴权方式，所以可以在多个服务器共享。

    缺点：由于Token需要解密，所以会比Session消耗更多的时间。

    SSO：单点登录

    # 前端请求方式

    请求方式 特点
    XHR JS 原生对象，封装程度低，调用麻烦
    Fetch ES6 内置的网络请求工具，支持 promise，但还是偏底层，需要再次封装使用
    Axios Axios 是一个基于 promise 的网络请求库，封装了很多常用的 api，操作方便，功能强大
