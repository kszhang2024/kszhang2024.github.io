    # vue 基本用法总结

    # 1、插值

    v-pre: 不解析插值内容，如{{msg}}在页面中渲染出来仍然是{{msg}}

    v-once: 仅渲染一次插值内容，后续不会修改插值 （如果显示的信息后续不需要再修改，使用v-once，这样可以提高性能。）

    v-text: 与{{}}插值一致，实现数据与页面的绑定。

    v-html: 如果插值内容中有html语法格式的内容，将插值内容中的html语法解析出来。

    # 2、v-bind

    简写模式：使用":"符号代替"v-bind:"。

    v-bind 主要是实现标签内的属性值与data或compute中数据的绑定，其中主要应用为class、style属性

    # 3、v-on

    1、$event 代表当前点击事件默认参数

    2、点击事件修饰符(可链式调用)：

    .stop 阻止事件冒泡

    .self 仅在该元素自身触发的时候才触发事件

    .capture 添加事件侦听器时，使用事件捕获模式（改变冒泡顺序）

    .prevent 阻止标签默认事件

    .once 事件仅触发一次

    # 4、v-if（v-else-if，v-else）和v-show

    v-if: 销毁dom节点 v-show: 修改节点样式为display：none，并不删除dom节点

    与display：none相似的为visibility：hidden 前者不会占据空间，后者会。

    # 5、v-for

    1、遍历数组（item，index），对象（item，key，index），

    2、key: 唯一标识（高效率）解决不唯一缓存问题

    不加key会导致每次数据改变时重新覆盖虚拟dom的原数据，再覆盖真实dom的数据

    加key可以使虚拟dom和真实dom仅改变改变的数据

    3、v-for` 也可以接受整数。在这种情况下，它会把模板重复对应次数。

    4、is属性：

     <ul>
    <li
      is="todo-item"
      v-for="(todo, index) in todos"
      v-bind:key="todo.id"
      v-bind:title="todo.title"
      v-on:remove="todos.splice(index, 1)"
    ></li>
  </ul>

    # 6、v-model

    1、与bind的区别：bind为单项绑定，model双向绑定（其实是v-bind:value="xxx"与v-on:input="xxx = $event.target.value"的语法糖）

    2、可以链式调用

    .lazy：懒加载，在每次结束输入时改变数据

    .number：将获取的数据转变为number类型

    trim：去除首尾空格

    3、复选框：单个值：v-model（boolean） 多个值：v-model（数组）

    # 7、组件

    # style：

    scoped 父子组件之间的样式重叠问题

    # 父子组件传值：

    1、父组件传递数据给子组件：

    利用props传递数据，不指定数据类型默认为字符串

    对象形式 {type指定类型，default指定默认值，required是否为必须属性}

    2、子组件传递数据给父组件（调用父组件函数）：

    $emit("父组件监听函数名", 函数参数)

    子组件通过$emit调用父组件函数来改变父组件数据达到子组件传递数据给父组件，如果父组件与子组件层级相差太大，利用vuex来管理

    # 父子组件方法互调：

    1、子组件调用父组件的方法：

    $parent:找到父组件

    $parent.count访问父组件数据

    $parent.changeNum()调用父组件方法

    parent 父组件的父组件

    $root: 访问根节点（最上层的节点）

    2、父组件调用子组件方法：

    $refs

    $refs.one.changNum()(先通过ref属性给子组件起别名，这里假定别名是one)

    $refs.one.count 也可以调用子组件数据

    # 8、 slot

    1、一个插槽也可以传入多个标签或嵌套标签。

    2、可以提供默认值，当不在父组件传递插槽内容的时候，会展示默认内容。

    3、也可以使用多个插槽，传递一个参数会替换所有插槽。

    4、具名插槽：

    ​ 直接传插槽内容替换未起名插槽.

    ​ 如果换掉起名插槽，使用template包裹插槽内容，v-slot标签（可简写为#，但有时会有问题）指定替换插槽名，未起名的插槽名为default

    ​ 父组件中使用子组件数据渲染插槽：

    ​ 子组件插槽：<slot name="插槽名" v-bind:user="user"></slot>

    ​ 父组件使用：<template v-slot:插槽名="任意名" >{{任意名.user}}</template>

    # 9、组件的生命周期

    # 10个生命周期：

    beforeCreate: 在实例初始化之后，数据观测 (data observer) 和 event/watcher 事件配置之前被调用。

    created: 在实例创建完成后被立即调用。在这一步，实例已完成以下的配置：数据观测 (data observer)，property 和方法的运算，watch/event 事件回调。然而，挂载阶段还没开始。

    beforeMount: 在挂载开始之前被调用：相关的 render 函数首次被调用。

    mounted: 实例被挂载后调用。

    beforeUpdate: 视图层数据更新前调用，发生在虚拟 DOM 打补丁之前。这里适合在更新之前访问现有的 DOM

    updated: 由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。

    beforeDestroy: 实例销毁之前调用。在这一步，实例仍然完全可用。

    destroyed: 实例销毁后调用。该钩子被调用后，对应Vue实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。

    activated: 被 keep-alive 缓存的组件激活时调用。

    deactivated: 被 keep-alive 缓存的组件停用时调用。

    # this.$nextTick(()=>{})

    将回调延迟到下次DOM更新循环之后执行，在修改数据之后立即使用它，然后等待DOM更新
