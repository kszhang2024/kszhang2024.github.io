    一文总结 vuex 基本用法

    ZCSheng 2021/11/13 Vue

    # Vuex（状态管理）：

    # 单向数据流示意：

    （actions响应view上的用户输入导致的状态变化，state是驱动应用的数据源，view以声明方式将state映射到视图）

    # 多个组件共享状态：

    （组件通过Actions向服务器端发起数据请求，服务器端返回数据，通过commit利用Mutations改变state中的数据，最后state的数据响应式的展示在组件上）

    # devtools

    vue官方提供的开发这工具，可以用来显示组件状态和状态变化。

    # State：

    Vuex中用来声明数据的地方，相当于Vue组件中的data，在组件中通过this.$store.state.数据名来获取数据，也可以在计算属性中来获取State中的数据。

    # Getters(Vuex中的计算属性)：

    类似于Vue中的计算属性，Vuex中也实现了自己的计算属性Getters，通过this.$store.getters.数据名获取数据。

    一个参数(state)：代表当前Vuex中的state。

    两个参数(state，getters)：state与上面一致，而getters则代表当前Vuex中的getters，有时候当前计算属性会依赖其他计算属性的值，就可以使用getters参数。

    如何传入用户自定义参数：由于getters不支持第三个参数，所以想要传入用户自定义的参数，可以通过返回函数的方式，相当于调用getters中的计算属性是调用了一个函数（此时每次都会去进行调用，而不会缓存结果）

    # Mutation：

    注意：请勿在这里使用异步方法，Vuex不知道回调函数何时被调用，导致无法追踪状态改变

    通过this.$store.commit('方法名', {name: "zhangsan"})提交状态改变，同时也可以this.$store.commit({type:'方法名',{name:'zhangsan'}})去提交状态改变，但需要注意此时payload为{type:'方法名',{name:'zhangsan'}}

    一个参数(state)：代表当前Vuex中的state。

    两个参数(state, payload)：state与上面一致，而payload则代表载荷（即用户传入的额外参数）,通过对象的方式传入，如果仅传入一个额外参数，也可以直接传参。

    # Action：

    由于Mutation中不能使用异步方法，而异步方法在数据请求中是无法避免的，所以在Action中你可以包含任意地异步操作，然后向Mutation提交状态。

    分发(dispatch)：

    在组件中通过this.$store.dispatch('方法名',{name: 'zhangsan'})调用action中的方法, 也可以通过this.$store.dispatch({type:'方法名',{name:'zhangsan'}})调用;

    首先，你需要明白 store.dispatch 可以处理被触发的action的处理函数返回的Promise，并且store.dispatch仍旧返回Promise

    一个参数(context)：context包含了state，getters，commit。

    可通过参数解构来简化代码({state, commit, getters})。

    两个参数(context,payload)：与Mutation相同，在Action里也提供了载荷来传递用户传入的额外参数。

    # Model：

    由于使用单一状态树，应用的所有状态会集中到一个比较大的对象，store对象会变得臃肿

    Vuex允许我们将store分割成模块（module）。每个模块拥有自己的state、mutation、action、getters，甚至是嵌套子模块——从上至下进行同样方式的分割。

    通过this.$store.state.user：访问user module中的数据

    mutation、getters、action尽量不要重名，因为子组件中的这些也是通过$this.store.commit("方法名")这些来调用的，所以为了避免指示不清楚，尽量不要重名。如果不能避免，可以使用命名空间解决。
