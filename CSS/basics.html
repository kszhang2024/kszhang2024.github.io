    # CSS 是如何工作的？

    首先，浏览器会加载HTML文件，对HTML文件进行解析。HTML文件会被解析为DOM树，然后创建DOM树。与此同时，DOM树中的CSS也会开始加载，经过一系列的计算过程，形成CSSOM。然后将CSSOM和DOM结合形成渲染树，对渲染树进行布局（Layout），最终呈现在页面上（着色）。

    # 伪类

    # 状态伪类

    名称 用途
    :link 选取未访问过的超链接元素。
    :visited 选取访问过的超链接元素。
    :hover 选取鼠标悬停的元素。
    :active 选取点中的元素。
    :focus 选取获得焦点的元素。

    注意：

    1. 在 CSS 定义中，a:hover 必须被置于 a:link 和 a:visited 之后，才是有效的。
    2. 在 CSS 定义中，a:active 必须被置于 a:hover 之后，才是有效的。

    # 结构性伪类

    名称 用途
    :first-child 选择父元素的第一个子元素。
    :last-child 选择父元素的最后一个子元素。
    :nth-child(an+b) 兄弟元素(序号 正序)中，满足表达式an+b匹配到的元素集合(n=0，1，2……)
    :nth-last-child(an+b) 兄弟元素(序号 倒序)中，满足表达式an+b匹配到的元素集合(n=0，1，2……)
    /* 匹配所有作为第一个子元素的 <p> 元素中的所有 <i> 元素 */
p:first-child i
{
    color:blue;
}

/* 匹配每一个序号为 6 的<p>元素 */
p:nth-child(6)
{
  background:#ff0000;
}

    # 常用属性总结

    颜色

    声明颜色的方式：

    1. RGB： 红色、绿色、蓝色(表达方式：1. 16进制 2. rgb函数)

    2. HSL：色相(0-360)、饱和度(0-100%)、亮度(0-100%)

    3. 关键字(red, green)

    alpha 透明度

    rgba(255，255，255，.5)：第四个参数不透明度(0 - 1) 1: 完全不透明 0: 透明

    #FF0000DB：八位，最后两位数字代表不透明度。

    属性 含义
    font-family serif：带衬线字体，笔画结尾有特殊的装饰线或衬线。
    sans-serif：无衬线字体，即笔画结尾是平滑的字体。
    cursive：草书字体。这种字体有的有连笔，有的还有特殊的斜体效果。
    monospace：等宽字体，即字体中每个字宽度相同。
    fantasy：主要是那些具有特殊艺术效果的字体。
    font-size 指定字体的大小。ex：x-small，medium，xx-large，larger，12px，80%
    font-style 指定字体的 italic 或 oblique 样式。ex：italic，oblique 20deg
    font-weight 指定了字体的粗细程度， 一些字体只提供 normal 和 bold 两种值。tips：normal与400等值。bold与700等值。
    line-height 指定元素行盒的最小高度。ex：normal，2.5，150%，32px
    letter-space 用于设置文本字符的间距。ex：normal，.2rem，1px，-1px
    word-space 用于设置单词的间距。
    text-align 定义行内内容如何相对它的块父元素对齐。ex：justify(两侧对齐，最后一行无效)
    text-indent 定义一个块元素首行文本内容之前的缩进量。
    属性 含义
    outline 绘制于元素周围的一条线，位于边框边缘的外围，可起到突出元素的作用。
    text-decoration 设置文本的修饰线外观的(none, underline, line-through, overline)
    white-space 设置如何处理元素中的空白(normal, nowrap, pre, pre-wrap, pre-line)
    vertical-align 指定行内元素或表格单元格元素的垂直对齐方式。(baseline, middle, top, bottom)

    font复合写法：

    font：斜体 粗细 大小/行高 字体族

    # 选择器的特异度（优先级）

    当一个标签满足多个选择器时，我们需要确定将哪一个选择器应用于该标签，此时选择器的特异度(优先级)就体现了他的作用。

    类型 优先级
    id 选择器 100
    (伪)类选择器 10
    伪元素、标签选择器 1

    # CSS 中的继承与初始值

    initial：指定该属性的值为默认值

    inherit：指定该属性的值从父模块继承

    unset：如果能继承，继承父元素的值，否则为默认值

    显示继承：子元素自动继承一些父元素属性，例如font-size...(文字相关属性继承，盒模型相关属性不继承)

    /* example */
* {
	box-sizing: inherit;
}

html {
    box-sizing: border-box;
}

.some-widget {
    box-sizing: content-box;

    # Layout

    确定内容的大小和位置的算法，依据元素、容器、兄弟节点和内容等信息来计算。

    定位的模式

    1. 常规流： 行级、块级、表格布局、Flex 布局、Grid 布局
    2. 浮动
    3. 绝对定位

    属性 含义
    width、height 指的是content box的宽度和高度。父容器有指定高度，height 百分数才生效。
    border border：border-width border-style border-color
    padding 内边距，一个参数(上下左右)、两个参数(上下、左右)、三个参数(上、左右、下)、四个参数(上、下、左、右)
    如果设置为百分比，始终相对于元素所在容器的宽度，即便是padding-top。
    margin 外边距。
    box-sizing 定义了 user agent 应该如何计算一个元素的总宽度和总高度。ex: content-box | border-box
    overflow 定义当一个元素的内容太大而无法适应块级格式化上下文的时候该做什么（visible，hidden，scroll、auto）

    利用 border 生成箭头：

    /* 将其他三个 border 设置为透明，保留需要的 border 颜色 */
.content {
    width: 0; 
    height: 0;
    border: 60px solid transparent;
    border-bottom-color: #00BBFF;
}

/* 注意：不能仅设置需要的 border */
.content {
    width: 0; 
    height: 0;
    border-bottom: 60px solid #00BBFF;

    overflow中scroll 、auto的区别：

    auto只会在内容溢出时显示滚动条。

    scroll始终显示滚动条，即使无内容溢出且此时不能滚动它。

    # Flex排版上下文

    属性 含义
    flex-direction 决定主轴的方向，row，row-reverse，column，column-reverse。
    justify-content 定义了项目在主轴上的对齐方式，flex-start，flex-end，center，space-between，space-around
    align-items 定义项目在交叉轴上如何对齐，flex-start，flex-end，center，stretch，baseline
    align-self 定义单个项目有与其他项目不一样的对齐方式，auto，flex-start，flex-end，center，baseline，stretch
    order 定义项目的排列顺序。数值越小，排列越靠前，默认为0
    flex flex-grow flex-shrink flex-basis
    flex-grow 定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大。
    flex-shrink 定义项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。
    flex-basis flex 元素在主轴方向上的初始大小。

    # float 浮动

    原来用于页面排版布局，后面由于flex、grid等更方便的排版方式出现，现在float已主要用于图文环绕效果的实现。

    # position 属性

    属性值 含义
    static 默认值，非定位元素。
    relative 在常规流里面布局，相对于自己本应该在的位置进行偏移，使用left、top、bottom、right 设置偏移距离，流内其他元素当它没有偏移一样布局。
    absolute 相对最近的非 static 祖先定位，脱离常规流，不会对流内元素布局造成影响。
    fixed 元素会被移出正常文档流，通过指定元素相对于屏幕视口（viewport）的位置来指定元素位置。元素的位置在屏幕滚动时不会改变
