    # 第一题 剑指 Offer 10- I. 斐波那契数列 (opens new window)

    写一个函数，输入 n ，求斐波那契（Fibonacci）数列的第 n 项（即 F(N)）。斐波那契数列的定义如下：

    F(0) = 0,   F(1) = 1
F(N) = F(N - 1) + F(N - 2), 其中 N > 1.
    1
    2

    斐波那契数列由 0 和 1 开始，之后的斐波那契数就是由之前的两数相加而得出。

    答案需要取模 1e9+7（1000000007），如计算初始结果为：1000000008，请返回 1。

    示例 1：

    输入：n = 2
输出：1
    1
    2

    示例 2：

    输入：n = 5
输出：5
    1
    2

    提示：

    • 0 <= n <= 100

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/fei-bo-na-qi-shu-lie-lcof

    解题思路：由于今天的题目标题为动态规划，于是看完题就想到了动态规划。可以按照常规的动态规划来分析，定义dp数组存储结果，遍历一次，利用F(N) = F(N - 1) + F(N - 2)计算当前答案，最后返回dp[n]。时间复杂度为O(n)，空间复杂度为O(n)。成功通过，然后开始看题解，发现可以仅记录F(N)、F(N - 1)、F(N - 2)三个值，如果n小于2，那么直接返回n。否则依次计算，返回F(N)。时间复杂度为O(n)，空间复杂度为O(1)。

    **注意：**为什么需要先在遍历的时候就取余，而不在最后返回的时候取余？因为有可能在计算过程中产生溢出，导致答案不正确。

    代码实现：

    var fib = function(n) {
    if (n < 2) {
        return n;
    }

    let pre2=0, pre1=1;
    let now;
    for (let i=2; i<=n; i++) {
        now = (pre2 + pre1) % (1e9 + 7);
        pre2 = pre1;
        pre1 = now;
    }

    return now;
};

    # 第二题 剑指 Offer 10- II. 青蛙跳台阶问题 (opens new window)

    一只青蛙一次可以跳上1级台阶，也可以跳上2级台阶。求该青蛙跳上一个 n 级的台阶总共有多少种跳法。

    答案需要取模 1e9+7（1000000007），如计算初始结果为：1000000008，请返回 1。

    示例 1：

    输入：n = 2
输出：2
    1
    2

    示例 2：

    输入：n = 7
输出：21
    1
    2

    示例 3：

    输入：n = 0
输出：1
    1
    2

    提示：

    • 0 <= n <= 100

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/qing-wa-tiao-tai-jie-wen-ti-lcof

    解题思路：看到这题，我们可以先列出递推方程，设台阶数为n，F(n)代表了n层台阶的跳法。

    F(n) = F(n-1) + F(n-2)

    F(0) = 1

    F(1) = 1

    看到递推方程后，发现递推方程和第一题是差不多的，只有初始值不同。那么第一题的代码稍作修改也能够通过。时间复杂度为O(n)，空间复杂度为O(1)。阅读题解后，发现题解其实也和目前思路差不多，多了一个矩阵快速幂的解法。

    代码实现：

    var numWays = function(n) {
    if (n < 2) {
        return 1;
    }

    let pre2=1, pre1=1;
    let now;
    for (let i=2; i<=n; i++) {
        now = (pre2 + pre1) % (1e9 + 7);
        pre2 = pre1;
        pre1 = now;
    }

    return now;
};

    # 第三题 剑指 Offer 63. 股票的最大利润 (opens new window)

    假设把某股票的价格按照时间先后顺序存储在数组中，请问买卖该股票一次可能获得的最大利润是多少？

    示例 1:

    输入: [7,1,5,3,6,4]
输出: 5
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。
     注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格。
    1
    2
    3
    4

    示例 2:

    输入: [7,6,4,3,1]
输出: 0
解释: 在这种情况下, 没有交易完成, 所以最大利润为 0。
    1
    2
    3

    限制：

    0 <= 数组长度 <= 10^5
    1

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/gu-piao-de-zui-da-li-run-lcof

    解题思路：因为前两道题都是动态规划解决的，所以看到这题时，又想用动态规划的思路来解决。最大利润应该是新增值和当前最小值的差、当前最大利润两个值之间的最大值，当数组长度为0或1的时候，最大利润为0。虽然通过了，但是执行效率很低，仅打败了33%的提交记录。现在，来看看其他人是怎么做的。大致思路是这样的，如果我们依次考虑每一天，假如当天我们要卖出股票，那么我们获得的最大利润应该是当天的值减去之前股票最低价买入的价格，然后每一天的最大利润中的最大值就是最大利润。

    代码实现：

    // 动态规划
var maxProfit = function(prices) {
    let dp = [];
    dp[0] = 0;
    dp[1] = 0;

    let min = prices[0];
    for(let i=1; i<prices.length; i++) {
        if(min > prices[i]) {
            min = prices[i];
        }
        dp[i+1] = Math.max(dp[i], prices[i] - min);
    }
    
    return dp[prices.length];
};

// 题解
var maxProfit = function(prices) {
    let minPrice = Infinity;
    let maxProfit = 0;

    for(let price of prices) {
        if (price < minPrice) {
            minPrice = price;
        }

        if(price - minPrice > maxProfit) {
            maxProfit = price - minPrice;
        }
    }

    return maxProfit;
};

    # 总结

    今天的题都和动态规划有关，动态规划常常适用于有重叠子问题和最优子结构 (opens new window)性质的问题。

    动态规划背后的基本思想非常简单。大致上，若要解一个给定问题，我们需要解其不同部分（即子问题），再根据子问题的解以得出原问题的解。如果有什么不对的地方，或者有更好的方案，欢迎提出，大家一起讨论。
