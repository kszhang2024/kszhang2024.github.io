    # 第一题 剑指 Offer 18. 删除链表的节点 (opens new window)

    给定单向链表的头指针和一个要删除的节点的值，定义一个函数删除该节点。返回删除后的链表的头节点。

    **注意：**此题对比原题有改动

    示例 1:

    输入: head = [4,5,1,9], val = 5
输出: [4,1,9]
解释: 给定你链表中值为 5 的第二个节点，那么在调用了你的函数之后，该链表应变为 4 -> 1 -> 9.

    2
    3

    示例 2:

    输入: head = [4,5,1,9], val = 1
输出: [4,5,9]
解释: 给定你链表中值为 1 的第三个节点，那么在调用了你的函数之后，该链表应变为 4 -> 5 -> 9.
    1
    2
    3

    说明：

    • 题目保证链表中节点的值互不相同
    • 若使用 C 或 C++ 语言，你不需要 free 或 delete 被删除的节点

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/shan-chu-lian-biao-de-jie-dian-lcof

    解题思路：不瞒大🔥，这类题我已经做过很多遍了。其中有一类题目给的参数是将要删除的指针，那么就可以将要删除的节点复制为后面那个节点，再删除后面那个节点，达到O(1)的时间复杂度。此题不同，参数给的是要删除的值以及头节点，那么正常删除即可，也不用双指针，先找到要删除节点的前一个节点，将其后面的节点删除一个就行。

    **注意：**如果要删除的是头节点，则需要判断单独处理，所以可以直接先加一个头节点，不用再判断。

    代码实现：

    var deleteNode = function(head, val) {
    // 添加头节点
    let newHead = new ListNode(0);
    newHead.next = head;
    node = newHead;

    while(node.next.val !== val) {
        node = node.next;
    }

    node.next = node.next.next;
    return newHead.next;
};

    # 第二题 剑指 Offer 22. 链表中倒数第k个节点 (opens new window)

    输入一个链表，输出该链表中倒数第k个节点。为了符合大多数人的习惯，本题从1开始计数，即链表的尾节点是倒数第1个节点。

    例如，一个链表有 6 个节点，从头节点开始，它们的值依次是 1、2、3、4、5、6。这个链表的倒数第 3 个节点是值为 4 的节点。

    示例：

    给定一个链表: 1->2->3->4->5, 和 k = 2.

返回链表 4->5.
    1
    2
    3

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/lian-biao-zhong-dao-shu-di-kge-jie-dian-lcof

    解题思路：这题很容易想到顺序查找的方法，先遍历一次链表，记录链表长度n，最后找到第n - k + 1个元素就是链表的倒数第k个元素，比如链表的长度为 10 ， 要找到倒数第 3 个元素，那我们可以去寻找第 10 - 3 + 1 = 8个元素(第 10 个元素为倒数第 1 个元素，第 9 个元素为倒数第 2 个元素)。

    但是但是，这种方法似不似太low了！！！

    接下来使用题解中的快慢指针方法，先让fast指针走到第 k 个元素，然后让slow指针和fast指针同步移动，当fast指针移动到链表末尾的时候，slow指针的位置就是倒数第 k 个位置。

    代码实现：

    // 快慢指针
var getKthFromEnd = function(head, k) {
    let fast = head;
    let slow = head;
	
    // fast 指针移动
    while(k) {
        fast = fast.next;
        k--;
    }
    
	// slow 指针移动
    while(fast) {
        slow = slow.next;
        fast = fast.next;
    }

    return slow;
};

    # 总结

    今天的题开启了双指针的篇章，但是在做今天第一题的时候并没有使用到双指针，也能够很好的完成题目，所以说不要盲目追求方法，而只要能更好的解决问题就行，毕竟方法也是为了解决问题而诞生的。对于第二题，接触到了一个新的概念——双指针中的快慢指针，其思想是一个指针快，一个指针慢，然后再同步移动。如果有什么不对的地方，或者有更好的方案，欢迎提出，大家一起讨论。
