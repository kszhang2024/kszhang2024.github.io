    # 第一题 剑指 Offer 25. 合并两个排序的链表 (opens new window)

    输入两个递增排序的链表，合并这两个链表并使新链表中的节点仍然是递增排序的。

    示例1：

    输入：1->2->4, 1->3->4
输出：1->1->2->3->4->4
    1
    2

    限制：

    0 <= 链表长度 <= 1000
    1

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/he-bing-liang-ge-pai-xu-de-lian-biao-lcof

    解题思路：声明新的链表头指针，如果l1中的值小于等于l2中的值，新头指针指向l1的值，l1指针后移，否则指向l2中的值，l2指针后移。

    **注意：**最后需要处理未遍历完的节点，将新链表的尾指针next指向未遍历完的链表即可，不用依次赋值。

    代码实现：

    var mergeTwoLists = function(l1, l2) {
   let mergeHead = new ListNode(0);
   let merge = mergeHead;

   while(l1 !== null && l2 !== null) {
        if(l1.val <= l2.val) {
            merge.next = l1;
            l1 = l1.next;
        } else {
            merge.next = l2;
            l2 = l2.next;
        }

        merge = merge.next;
   }
   
   merge.next = l1 === null ? l2 : l1;
   return mergeHead.next;
};

    # 第二题 剑指 Offer 52. 两个链表的第一个公共节点 (opens new window)

    输入两个链表，找出它们的第一个公共节点。

    如下面的两个链表：

    在节点 c1 开始相交。

    示例 1：

    输入：intersectVal = 8, listA = [4,1,8,4,5], listB = [5,0,1,8,4,5], skipA = 2, skipB = 3
输出：Reference of the node with value = 8
输入解释：相交节点的值为 8 （注意，如果两个列表相交则不能为 0）。从各自的表头开始算起，链表 A 为 [4,1,8,4,5]，链表 B 为 [5,0,1,8,4,5]。在 A 中，相交节点前有 2 个节点；在 B 中，相交节点前有 3 个节点。

    输入：intersectVal = 2, listA = [0,9,1,2,4], listB = [3,2,4], skipA = 3, skipB = 1

输入解释：相交节点的值为 2 （注意，如果两个列表相交则不能为 0）。从各自的表头开始算起，链表 A 为 [0,9,1,2,4]，链表 B 为 [3,2,4]。在 A 中，相交节点前有 3 个节点；在 B 中，相交节点前有 1 个节点。

    示例 3：

输出：null

    注意：

    • 如果两个链表没有交点，返回 null.
    • 在返回结果后，两个链表仍须保持原有的结构。
    • 可假定整个链表结构中没有循环。
    • 程序尽量满足 O(n) 时间复杂度，且仅用 O(1) 内存。
    • 本题与主站 160 题相同：https://leetcode-cn.com/problems/intersection-of-two-linked-lists/

    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/liang-ge-lian-biao-de-di-yi-ge-gong-gong-jie-dian-lcof

    解题思路：看了这题之后，想了很久都没想出来，感觉像判断两个数组中的公共部分，那么暴力法就用双重循环去遍历每一个元素，直到遇到相同的元素，再去同步遍历剩下的元素，看是否相等。但是最后太复杂了，没能实现。翻阅题解后，才发现有两种解法，并且自己对题的理解有一些偏差，后面的节点竟然连地址都是一样的，所以说，我们只需要找到两个地址相同的元素。所以，双重循环应该能做到了，相当于去寻找两个相等的元素。可是，这种解法时间复杂度太高，我们应该寻找更优的算法。

    哈希表

    遍历第一个链表，将节点存储起来，然后遍历第二个链表，看哈希表中是否含有当前节点。如果有，就说明找到了公共节点。找到就返回该节点，否则返回null。

    双指针

    这个解法是一个难以想到的点，在此引用推荐题解中的话来解释。

    双指针解法 (opens new window)

    代码实现：

    var getIntersectionNode = function(headA, headB) {
    if(headA === null || headB === null) {
        return null;
    }

    let pa = headA, pb = headB;
    while(pa !== pb) {
        pa = pa === null ? headB : pa.next;
        pb = pb === null ? headA : pb.next;
    }

    return pb;
};
