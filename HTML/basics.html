    # HTML

    # “<!DOCTYPE html>"

    文档类型声明，确保浏览器按照最佳的相关规范进行渲染，防止浏览器在渲染文档时，切换到我们称为**怪异模式（兼容模式）**的渲染模式。

    # 究竟什么是DOM

    DOM的中文含义是文档对象模型。浏览器根据文档内容而构建出来的树形结构，就被称为DOM树。在JS中，提供了一系列操作DOM的API。

    为什么需要DOM？

    1. 当JS需要修改页面内容时，直接更改文档文本是不方便的。

    2. 浏览器直接处理修改的文档文本也不方便，很难找到JS修改的地方，只能重新解析整个文档文本。这样不仅仅性能低，带来的用户体验也不友好，会出现白屏或闪屏等问题。

    浏览器根据DOM树实现页面的渲染，JS直接操作DOM树。这样JS就可以便捷的以类似操作对象的方式来修改页面。浏览器在这样的工作方式下能够找到修改的节点，然后重新渲染被修改影响的局部页面就可以了(回流或重绘)。

    # <meta>标签

    <meta>标签提供的信息对于用户而言是不可见的，但在网页被搜索时，起到了关键性作用，决定了网页在搜索中的排名。

    <meta>标签利用name和http-equiv两个属性设置属性名，content设置属性值(必需)。

    http-equiv：该属性用于设置与http响应头相关的属性

    <!-- 设置浏览器的过期时间 -->
<meta http-equiv="expires" content="16 Jan 2022">
<!-- 设置5秒自动刷新，跳转指定网页，未指定url则为当前页面 -->
<meta http-equiv="refresh" content="5 url=http://www.baidu.com">
<!-- 禁止浏览器从本地计算机的缓存访问页面 -->
<meta http-equiv="pragma" content="no-cache">

    # <script>标签

    <script>标签的作用有两个：1. 引入可执行脚本 2. 嵌入可执行脚本

    <script>加载过程：默认情况下，浏览器是同步加载 JavaScript 脚本，即渲染引擎遇到<script>标签就会停下来，等到执行完脚本，再继续向下渲染。如果是外部脚本，还必须加入脚本下载的时间。

    如果要使用异步加载，可以利用defer和async两个属性。二者都会异步加载，那么这两个属性的区别在哪儿呢？defer是当浏览器执行完渲染时，再执行，脚本的执行顺序会按照声明的顺序执行。async是当脚本下载完就会中断渲染，执行脚本，执行顺序不一定会按照声明的顺序。

    如果要使用ES6模块，可以加入type="module"属性

    <script type="module">
    import utils from "..."
</script>

    # 常用标签

    标题：h1 ~ h6

    列表：

    ol：有序列表，子元素：li

    ul：无序列表，子元素：li

    dl：key-value（定义列表），dt(标题)，dd(描述)

    <dl>
   <dt>计算机</dt>
   <dd>用来计算的仪器 ... ...</dd>
   <dt>显示器</dt>
   <dd>以视觉方式显示信息的装置 ... ...</dd>
</dl>

    链接：

    a：

    1. href：超链接地址。

    2. target：

      • _blank：在新窗口中打开被链接文档。

      • _top：在整个窗口中打开被链接文档。

      • _parent：在父框架集中打开被链接文档。

      • _self：默认。在相同的框架中打开被链接文档

    多媒体：

    img（图片）：alt：替代文本，width：宽度

    audio（音频）：src：音频路径，control：控制器

    在浏览器不支持时，会显示<audio></audio>标签之中的内容

    video（视频）：src：视频路径，control：控制器

    也可利用 source 标签提供音频、视频路径。浏览器选用支持的第一个 source 提供的路径。

    <video controls width="250">
    <source src="/media/flower.webm">
    <source src="/media/flower.mp4">
    Sorry, your browser doesn't support embedded videos.
</video>

    # HTML 中的语义化

    利用合适的标签来描述合适的内容，使元素、属性、属性值都有某些含义。

    语义化标签示例

    header(nav)：页面头部

    main(article)：主要内容

    aside：与主要内容关系不大的内容，如广告

    footer：页面尾部

    section：表示一个包含在HTML文档中的独立部分

        <header>
        <h1>这是标题</h1>
        <nav>
            <ul>
                <li><a href="#first">第一部分</a></li>
                <li><a href="#second">第二部分</a></li>
            </ul>
        </nav>
    </header>
    <main>
        <aside>广告</aside>
        <section id="first">
            <h2>这是第一部分</h2>
            <article><h3>文章标题 11</h3><p>文章内容 11</p></article>
            <article><h3>文章标题 12</h3><p>文章内容 12</p></article>
        </section>
        <section id="second">
            <h2>这是第二部分</h2>
            <article><h3>文章标题 21</h3><p>文章内容 21</p></article>
            <article><h3>文章标题 22</h3><p>文章内容 22</p></article>
        </section>
    </main>

    <footer>
        <p>author: ZCSheng</p>
        <address>China</address>
        <p>2022.1.16</p>
    </footer>

    为什么要语义化？

    1. 提高代码可维护性。
    2. 便于浏览器识别。
    3. 利于搜索引擎优化。
    4. 保证程序在各个情况下可运行，例如，盲人阅读器的识别，如果用户看不见文本，那么使用阅读器也能够识别文本内容。

    如何做到语义化？

    1. 了解每个标签和属性的含义

    2. 思考什么标签最适合描述这个内容
