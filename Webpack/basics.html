    # 什么是 Webpack？

    在前端工程化之前，手动引入css、js、jpg，svg等前端资源。操作十分复杂，同时不方便扩展less，ts等工具，开发效率低。为了解决这些问题，逐渐出现了 "前端工程化" 的概念。并出现了大量工具，webpack 就是其中一种。

    Webpack 本质上 就是一种前端资源编译、打包工具。（模块化 + 一致性）

    # 如何使用 Webpack？

    1. 安装依赖
    npm i -D webpack webpack-cli

    webpack v4.0.0 及以上 支持 0 配置使用，不需要编写配置文件。

  mode: 'development', // 模式
  entry: 'main.js', // 打包入口地址
  output: {
    filename: '[name].js', // 输出文件名
    path: path.join(__dirname, './dist') // 输出文件目录
  },
};

    # 基础配置

    使用 webpack 时，最关键的点就是配置文件了，我们先来看看一些常用的基础配置。

    # mode（模式）

    表示 webpack 当前运行的环境，自动调用 DefinePlugin 插件添加全局变量 process.env.NODE_ENV 为指定值。

    • production(默认)，生产模式，打包比较慢，会开启 tree-shaking 和 压缩代码
    • development，开发模式，打包更加快速，省了代码优化步骤
    • none，不使用任何默认优化选项

    # context

    用来指定相对路径的根目录，必须为绝对路径，默认值为启动 Webpack 时所在的当前工作目录。

    const path = require('path');

module.exports = {
  context: path.resolve(__dirname, 'src'),
};

    # entry

    entry 是配置模块的入口。entry 的值可以是string，array，object，均可以使用相对路径。

    string和array代表入口模块的文件路径，而array则是代表将多个入口文件打包在一起。二者均只会生成一个名称为main的chunk。

    module.exports = {
  entry: './file.js',
};

/* 上面是下面的简写形式 */
module.exports = {
  entry: {
    main: './file.js',
  },
};

    # output

    output 指定了 webpack 如何输出编译文件，其值为一个object。

    属性 含义
    filename 指定输出文件名称
    path 指定输出文件的目录
    publicPath 资源引入公共路径前缀，一般用于生产环境
    const path = require('path');

/* 只有一个输出文件 */
module.exports = {
  entry: './src/main.js',
  output: {
    filename: 'main.js',
    path: path.resolve(__dirname, 'dist'),
  },
};

/* 多输出文件，利用内置变量 name 动态创建名字 */
/* 输出 ./dist/main.js, ./dist/other.js */
module.exports = {
  entry: {
    main: './src/main.js',
    other: './src/other.js',
  },
  output: {
    filename: '[name].js', // 这里 name 的值就是入口文件的 chunk 名称
    path: path.resolve(__dirname, 'dist'),
  },
};

    # 理解 Loader

    webpack 本身只能处理js和json文件。loader 使 webpack 能够处理其他类型的文件，并将它们转换为有效模块，供应用程序使用。

    loader 是从后往前执行的。loader 支持异步执行、链式执行。

    loader 有两个属性：

    • test：识别出哪些文件会被转换。

    • use：定义在进行转换时，应该使用哪个 loader。

    # 常用 Loader

    postcss-loader

    自动添加 CSS3 部分属性的不同浏览器前缀。

    使用方式：

    npm install postcss postcss-loader postcss-preset-env -D
    1

    创建配置文件：postcss.config.js（postcss 配置文件）、.browserslistrc（postcss-preset-env 配置文件）

    module.exports = {
	...
	module: {
	  rules: [
	    // 处理sass，scss，css
	    // npm install sass-loader -D
	    // npm i node-sass --sass_binary_site=https://npm.taobao.org/mirrors/node-sass/
	    {
	      test: /\.(s[ac]|c)ss$/i, // scss, sass, css 文件，i 忽略大小写
	      use: [
              // 'style-loader',	style-loader 通过嵌入的方式加载css
              MiniCssExtractPlugin.loader, // 添加 loader，我们通过插件以外链形式加载css
              'css-loader',
              'postcss-loader',
              'sass-loader'
          ]
	    },
	    // 处理图片
	    {
           test: /\.(jpe?g|png|gif)$/i, // jpg、jpeg、png、gif 文件
           use: [
             // 'file-loader'	不会对图片进行处理，仅拷贝图片到指定目录
             {
               loader: 'url-loader',
               options: {
                 name: '[name][hash:8].[ext]',
                 limit: 10 * 1024	 // 文件小于 10k 会转换为 base64，大于则拷贝文件
               }
             }
           ]
        },
	    // 处理Vue SFC文件
	    {
	      test: /\.vue$/,
	      use: 'vue-loader'
	    },
	    // 处理字体
	    {
	      test: /\.(woff2?|eot|ttf|otf)(\?.*)?$/i,
	      use: [
	        {
	          loader: 'url-loader',
	          options: {
	            name: 'fonts/[name][hash:8].[ext]',	// 体积大于 10KB 打包到 fonts 目录下 
	            limit: 10 * 1024,
	          }
	        }
	      ]
	    },
	    // 引入 ts
	    {
	      test: /\.tsx?$/,
	      use: 'ts-loader'
	    },
		// 引入 babel
		// npm install babel-loader @babel/core @babel/preset-env -D
		// babel-loader 使用 Babel 加载 ES2015+ 代码并将其转换为 ES5
		// @babel/core Babel 编译的核心包
		// @babel/preset-env Babel 编译的预设，可以理解为 Babel 插件的超集
		{
		  test: /\.js$/,
		  exclude: /node_modules/,	// 不需要转化的代码
		  use: {
		  	loader: 'babel-loader',
		  	options: {
		  	  presets: ['@babel/preset-env'],
		  	}
		  }
		},
	  ]
	},
    plugins: [
      ...
      new MiniCssExtractPlugin({ // 添加mini-css-extract-plugin插件
        filename: '[name].[hash:8].css'
      }),
      ...
    ]
    ...
}

    # 如何实现 Loader

    Loader 其实就是一个接受三个参数的函数，当我们需要手写 Loader 时，需要在该函数中实现代码逻辑，然后在配置文件中引入该函数即可。

    // 样例 Loader 实现
/* exLoader.js */
/*
  source: loader 的输入内容，可能是文件内容，也可能是上一个 loader 处理结果
  sourceMap: 代码的 sourcemap 结构
  data: 其他需要传递的信息
*/
module.exports = function(source, sourceMap?, data?) {
  return source;
};

/* webpack.config.js */
module.exports = {
  ...
  resolveLoader: {
      // 规定 Loader 加载路径，从右到左
      modules: ['node_modules', './']
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        use: 'exLoader'
      }
    ]
  }
  ...
}

    # 理解 Plugin

    loader 用于转换某些特定类型的模块，是 Webpack 的翻译官。插件是用来扩展 Webpack 功能的，它的执行贯穿整个 Webpack 的生命周期，可以用于执行比 loader 范围更广的任务：打包优化、资源管理、注入环境变量等。

    # 常用 Plugin

    Plugin 作用
    html-webpack-plugin 自动生成 html 文件，并将打包生成的资源引入html中
    define-plugin 定义环境变量（指定mode后自动调用该插件）
    mini-css-extract-plugin 从css文件中提取css代码到单独的文件中，对css代码进行代码压缩等
    clean-webpack-plugin 自动清除之前打包的文件
    webpack-parallel-uglify-plugin 并行的进行代码压缩。

    # 如何实现 Plugin

    Plugin 的实现比 Loader 要稍难一点。它其实是一个类，在它的apply方法中，我们可通过各种compiler.hooks来实现我们的代码逻辑。最后我们在 webpack.config.js 的 Plugin中引用自己编写的 Plugin 就行。

    # HMR（模块热替换）

    HMR 是用来提高生产效率的工具，在修改并保存代码后，webpack 对代码重新进行打包，发送到浏览器，浏览器用新模块替换旧模块，在不刷新页面的情况下更新应用。

    /* 启用 HMR */
module.exports = {
  ...
  devServer: {
    contentBase: "./src", // 设置服务器访问的基本路径
    hot: true,
    open: true, // 自动打开页面
  },
  ...
}

    # Source Map

    从已转换的代码映射到原始源的文件，使浏览器能够重构原始源并在调试器中显示重建的原始源。

    启用Source Map的方式是配置 devtool，不同的值有不同的生成方式，但其值大多数都是由5个基础值组合起来的。

    基础值 含义 作用
    source-map 产生 .map 文件 定位信息最全，但也.map 文件最大，效率最低
    eval 使用 eval 包裹块代码 利用字符串可缓存从而提效
    cheap 不生成列信息 精准度降低换取文件内容的缩小
    module 包含 loader 的 source map 解决对于使用``cheap 配置项导致无法定位到 loader 处理前的源代码问题
    inline 将 .map 作为 DataURI 嵌入，不单独生成一个 .map 文件 减少文件数
    /* 开发环境常用 */
devtool: 'cheap-module-eval-source-map'

/* 生产环境常用 */
devtool: 'hidden-source-map'

    # Tree-Shaking

    Tree-Shaking 是用来优化 webpack 打包的工具，用来删除无用的js代码。由于 Tree-Shaking 是基于 ESM 的，所以引用的文件需要遵循 ES 模块标准。

    /* 启用 Tree-Shaking */
module.exports = {
	entry: './src/main.js',
	mode: 'production',
	output: {
	  path: path.resolve(__dirname, 'dist'),
	  filename: "main.js",
	},
    optimization: {
      usedExports: true,
    },
}
